# Deploy Checklist

Короткий чеклист перед каждым пушем в `main`, чтобы изменения точно попадали на прод.

## Почему пропускаются изменения

Обычно это происходит по одной из причин:
- изменения внесены в открытом файле, но файл не сохранен;
- в коммит попали не все нужные файлы (`git add` был слишком узким);
- после пуша не проверили, что прод отдает новую версию файла.

## Обязательные шаги перед пушем

1. Сохранить все файлы в IDE.
2. Проверить diff:
   - `git status --short`
   - `git diff -- js/annotations.js`
3. Убедиться, что нужные файлы staged:
   - `git add js/annotations.js js/app.js index.html`
   - `git diff --cached --name-only`
4. Сделать коммит и пуш:
   - `git commit -m "..." && git pull --rebase && git push`

## Обязательная проверка после деплоя (чтобы не было «ничего не вижу на проде»)

1. **Проверить, что прод отдаёт актуальные файлы:**
   ```bash
   curl -s "https://frenchb22026.bogachev.fr/js/app.js" | head -5
   curl -s "https://frenchb22026.bogachev.fr/js/annotations.js"
   ```
   Если в ответе нет последних правок — они не попали в коммит/пуш.

2. **Проверить, что сайт реально работает (не пустой экран):**
   - Открыть в браузере: https://frenchb22026.bogachev.fr
   - Убедиться, что виден дашборд (заголовок, KPI, графики или «Loading…»), а не белый/пустой экран.
   - Открыть DevTools (F12) → вкладка Console: не должно быть красных ошибок (Uncaught TypeError, etc.).
   Если экран пустой или в консоли ошибка — деплой считаем неуспешным, фиксим код и пушим снова.

3. **Почему бывает пустой экран:** ошибка в JS до или при первом рендере (например деление на ноль в расчёте стилей, обращение к undefined). После пуша всегда делать шаги 1 и 2.

## Практика для аннотаций

- Всегда править только `js/annotations.js`.
- После редактирования сразу выполнять:
  - `git diff -- js/annotations.js`
  - `curl -s "https://frenchb22026.bogachev.fr/js/annotations.js"` (после пуша)
